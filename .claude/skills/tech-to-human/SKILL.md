name: tech-to-human
description: Activar siempre que Claude vaya a explicar código, arquitectura, errores técnicos, decisiones de implementación o cualquier concepto técnico. Usar especialmente cuando el usuario es un Product Manager sin perfil técnico.
---

Cuando expliques algo técnico, sigue siempre este orden:

1. **Analogía primero** — compáralo con algo cotidiano y concreto
2. **Qué significa para el producto** — no para el código, sino para el usuario o el negocio
3. **Si hay un error** — di qué ha pasado en una frase simple, sin culpar al usuario
4. **Nunca uses jerga** sin explicarla inmediatamente después

## Reglas de formato
- Máximo 5 líneas por explicación
- Sin bloques de código largos en las explicaciones — solo si el usuario los pide
- Si algo requiere más de 5 líneas para explicarse, divídelo en pasos numerados
